# Contributing to Zero to LPT Certified

Thanks for helping make this the best open CPENT/LPT study resource.

## Ground rules

1. **No exam content.** Never submit real CPENT/LPT questions, answers, flag values, or
   range details. Doing so violates the EC-Council Candidate Agreement and will be
   rejected. We teach *concepts and methodology*, not dumps.
2. **Legal/ethical only.** All techniques must be framed for authorized testing and
   self-built labs.
3. **Cite sources** for any factual claim about the exam (format, scoring, domains).
4. **Test your scripts** before submitting. Note the OS/version you tested on.

## How to contribute

- Fork → branch (`feat/<topic>` or `fix/<topic>`) → PR against `main`.
- Keep Markdown clean (headings, fenced code blocks with language hints).
- One logical change per PR; describe *what* and *why*.

## What we especially want

- Additional vulnerable lab targets and build scripts.
- Cheatsheet one-liners (with a one-line explanation each).
- Corrections/updates when EC-Council revises the blueprint.
- Better diagrams for the lab network.

## Style

- Commands in fenced blocks with the right language (`bash`, `powershell`, `text`).
- Prefer free/open tools. If a tool is commercial (e.g., Burp Pro), note the free alternative.
